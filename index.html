CoyoteCert

Modern ACME for Modern PHP

Look, you need TLS certs. You don't need another weekend reading RFC 8555. CoyoteCert is a PHP 8.3+ client that talks ACME to Let's Encrypt, ZeroSSL, Buypass, and anyone else with a compliant directory. It issues. It renews. It revokes. It does not explode at 3am.

§ 7.4
№ 01

RFC 8555, by the book

Accounts, orders, authorizations, finalization. Nonces, replay protection, JWS signing. All the boring plumbing happens inside the library. You just ask for a cert.

HTTP + DNS
№ 02

HTTP-01 and DNS-01

HTTP challenge? Built in. DNS-01 for wildcards? Cloudflare, Route 53, DigitalOcean already packed in. Running something weirder? Two methods to implement and you're certified.

TYPED
№ 03

Types, not mystery arrays

PHP 8.3 readonly classes. Enums for every state. Return types everywhere. Your IDE knows what it's doing, and so will you. No more printing a response and squinting at it.

KEYS
№ 04

ECDSA or RSA, your call

P-256, P-384, P-521 or honest 2048 to 4096 bit RSA. Store them on disk, in Vault, in KMS, in a cigar box. One interface, one method, bring whatever you already trust.

AUTO
№ 05

Renewal on autopilot

One cron line. vendor/bin/coyotecert renew --due. Idempotent, atomic, skips what isn't due. No PHP script. No drama. Run it every hour if you feel fancy. The CA won't even notice.

CLI
№ 07

Embedded CLI, no PHP required

Not a PHP person? Nobody's judging. vendor/bin/coyotecert issue example.com. Cert issued, go touch grass. Cron, CI, Docker, a shell script from 2009 — it runs wherever shells run.

OBSERVE
№ 06

PSR-3 logs, zero mystery

Every request, challenge poll, and retry goes through a logger you hand us. When something goes sideways you get receipts. When nothing goes sideways you also get receipts.

Issue a cert

One chain. One cert.
Zero anvils.

Point it at a CA. Hand it a domain and a solver. Get back a fullchain and a private key. That's the whole show. The paperwork is hiding in the basement where it belongs.

  • ✓Hit Let's Encrypt staging first. Then prod. Same code, different URL.
  • ✓Swap CAs by changing one string. ZeroSSL, Buypass, SSL.com, step-ca.
  • ✓Get a real Certificate object. Not a sack of PEM you have to shake out.
  • ✓Typed exceptions for rate limits, bad nonces, and other CA tantrums.
IssueCertificate.php PHP · 8.3 
use Blendbyte\CoyoteCert\CoyoteCert;
use Blendbyte\CoyoteCert\Provider\LetsEncrypt;
use Blendbyte\CoyoteCert\Storage\FilesystemStorage;
use Blendbyte\CoyoteCert\Challenge\CloudflareDns01Handler;

$cert = CoyoteCert::with(new LetsEncrypt())
    ->storage(new FilesystemStorage('/var/certs'))
    ->identifiers(['example.com', '*.example.com'])
    ->email('admin@example.com')
    ->challenge(new CloudflareDns01Handler(apiToken: 'your-api-token'))
    ->issueOrRenew();

Three steps, tops

Install. Issue. Forget.

STEP 01

Grab the package

Composer, the way nature intended. No weird PECL extensions. Just OpenSSL.

$ composer require blendbyte/coyotecert
STEP 02

Catch your first cert

Test on staging. Flip to prod when it stops crashing. (It won't.)

CoyoteCert::with(new LetsEncrypt())->issueOrRenew();
STEP 03

Set it, forget it

Skip the PHP script. Slap the CLI in cron. Done.

0 * * * * vendor/bin/coyotecert renew --due
